# Roadmap и приоритеты реализации

Детальный план разработки с учётом AI/LLM-ускорения и стратегии выхода на рынок.

---

## Принципы приоритизации

1. **MVP за неделю** — используем AI/LLM для максимальной скорости
2. **Внутренний InSourceHub** — первые пользователи внутри банка
3. **Открытый код сразу** — прозрачность и рост сообщества
4. **Playwright first** — самый популярный современный фреймворк
5. **Запросы пользователей** — развитие по реальным needs, не гипотезам

---

## MVP Неделя 1: Core + Playwright

**Цель:** Работающий self-healing для Playwright на одном сценарии

| Приоритет | Фича | Задачи | Результат |
|-----------|------|--------|-----------|
| P0 | F001 Core | Rust core: сигнатуры DOM, кластеризация | WASM библиотека для сравнения элементов |
| P0 | F008 Adapter | Playwright custom selector `fletta:` | Плагин для Playwright на npm |
| P0 | C001 Demo | Сценарий Payment Button | Тест проходит после смены data-testid |

**Чек-лист успеха MVP:** (полный список gates: [benchmark.md](./benchmark.md))

- [ ] `npm install @fletta/playwright` работает
- [ ] Код на GitHub с README + [positioning.md](./positioning.md) one-liner
- [ ] **CP001–CP003** проходят на demo app (0 false positive на CP003)
- [ ] **CP005** — JUnit/артефакт в CI
- [ ] C001 проходит (демо для людей)
- [ ] Карточка **InSourceHub** ([audience.md](./audience.md))

**Время:** 5-7 дней с AI/LLM-помощью
