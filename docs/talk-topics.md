## T002: От Selenium к Self-Healing: миграция без боли

**Формат:** Доклад 20-30 минут + live demo

**Для кого:** Команды с legacy-тестами, QA automation engineers

**Конференции:** SeleniumConf, QA Guild, внутренние митапы банков

**Ключевые тезисы:**
- Playwright adapter — используем fletta без переписывания
- Постепенная миграция: замена селекторов по одному
- ROI: сколько времени экономим на рефакторинге
- Интеграция с CI: GitHub Actions, Jenkins

**Live demo:**
1. Берём существующий тест на Playwright (100 строк)
2. Меняем data-testid в приложении
3. Тест падает → добавляем fletta adapter
4. Тест проходит без изменения кода

**Что должно быть готово:**
- Playwright adapter в production
- Документация по миграции
- 2-3 истории успеха от банков/enterprise

## T004: Root Cause Analysis: когда тест падает не из-за UI

**Формат:** Воркшоп 60-90 минут (интерактивный)

**Для кого:** QA engineers, SRE, DevOps

**Конференции:** DevOops, SREconf, внутренние тренинги

**Программа:**
1. Типы падений: UI-изменение vs API vs инфраструктура vs race condition
2. Unified Context: объединение UI, логов, сети
3. Timeline анализ: что происходило за 5 секунд до падения
4. Практикум: разбираем 3 реальных падения, находим причину
5. Автоматический RCA-репорт для CI/CD

**Что должно быть готово:**
- Unified Context полностью работает
- Дашборд для визуализации timeline
- 5+ подготовленных кейсов для воркшопа

## T008: Тестируем AI-агентов: от MCP к A2A

**Формат:** Доклад 30-40 минут (трендовый, перспективный)

**Для кого:** AI-инженеры, разработчики агентов, QA для AI-систем

**Конференции:** AI Journey, CodeFest, конференции по LLM/agents

**Ключевые тезисы:**
- Почему AI-агенты тоже нуждаются в тестировании
- Capture agent sessions: что записываем, как анализируем
- Testing MCP tools: аргументы, результаты, side effects
- A2A flow testing: координация между агентами
- Prompt regression: тестируем при смене модели
- Compliance и аудит: что делал агент, почему

**Демо:**
1. AI-агент тестирует приложение через fletta MCP
2. fletta анализирует что и как агент вызывал
3. Смена модели GPT-4 → Claude — проверяем behavior
4. 2 агента через A2A — тестируем координацию

**Что должно быть готово:**
- F011 работает в production
- 2-3 реальных сценария тестирования агентов
- Сравнение моделей на одном и том же наборе задач
