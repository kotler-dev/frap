# Demo Case: CDP Recording & Scenario Playback (C009)

## Meta

- **ID**: C009
- **Title**: CDP Recording & Scenario Playback
- **Features**: F001, F004, F008
- **Status**: concept
- **Created**: 2026-05-20

## Goal

Демонстрация полного цикла: запись сценария через Chrome DevTools Protocol → генерация Page Object → воспроизведение с self-healing.

## Setup

### Prerequisites

```bash
# Запуск демо-приложения
cd demo-app
docker-compose up -d checkout-app

# Проверка доступности
open http://localhost:3000/checkout
```

### Файлы для кейса

```
demo-app/
├── checkout-v1/          # Базовая версия
├── checkout-v2/          # С рефакторингом (сменили testid)
└── checkout-ambiguous/   # Две похожие кнопки
```

## Steps

### Step 1: Запись сценария

```bash
# Запускаем запись с CDP
frap record \
  --url http://localhost:3000/checkout \
  --output ./scenarios/payment-flow.yml \
  --name "payment-flow"
```

Пользователь проходит сценарий в браузере:
1. Вводит email в поле
2. Выбирает способ оплаты
3. Нажимает "Pay Now"
4. Подтверждает заказ

### Step 2: Проверка записанного сценария

```bash
# Сценарий сохраняется в YAML
cat ./scenarios/payment-flow.yml
```

Ожидаемая структура:

```yaml
scenarioId: "payment-flow"
startUrl: "http://localhost:3000/checkout"
timestamp: "2024-01-20T10:30:00Z"

recordedActions:
  - id: "action-1"
    type: "navigate"
    url: "http://localhost:3000/checkout"
    
  - id: "action-2"
    type: "fill"
    element:
      logicalName: "emailInput"
      selectors:
        primary: "[data-testid='email-field']"
        alternatives:
          - "input[type='email']"
          - "//input[@name='email']"
    value: "user@example.com"
    
  - id: "action-3"
    type: "click"
    element:
      logicalName: "paymentMethodCard"
      selectors:
        primary: "[data-testid='pay-card']"
        alternatives:
          - "input[value='card']"
          
  - id: "action-4"
    type: "click"
    element:
      logicalName: "payButton"
      selectors:
        primary: "[data-testid='pay-btn']"
        alternatives:
          - "button[type='submit']"
          - "//button[contains(text(), 'Pay')]"

generatedPageObjects:
  - className: "CheckoutPage"
    elements:
      - name: "emailInput"
        type: "TEXT_INPUT"
        selectors:
          - "[data-testid='email-field']"
          - "input[type='email']"
          
      - name: "payButton"
        type: "SUBMIT_BUTTON"
        selectors:
          - "[data-testid='pay-btn']"
          - "button[type='submit']"

      - name: "fillEmail"
        parameters: ["email: String"]
        body: "emailInput.fill(email);"
        
      - name: "clickPay"
        body: "payButton.click();"
```

### Step 3: Воспроизведение сценария

```bash
# Базовый запуск — должен пройти без healing
frap replay \
  --scenario ./scenarios/payment-flow.yml \
  --url http://localhost:3000/checkout
```

Ожидаемый результат:
```
✓ PASSED
  healed: false
  actions: 4/4
  duration: 3.2s
```

### Step 4: Воспроизведение после рефакторинга

```bash
# Переключаемся на версию с изменёнными селекторами
docker-compose up -d checkout-v2

# Запускаем тот же сценарий
frap replay \
  --scenario ./scenarios/payment-flow.yml \
  --url http://localhost:3000/checkout \
  --min-confidence 0.85
```

Ожидаемый результат:
```
✓ PASSED (healed)
  healed: true
  healing_events: 2
  confidence: [0.92, 0.89]
  actions: 4/4
  duration: 4.1s
```

Отчёт о healing:
```yaml
healing_report:
  - actionId: "action-2"
    original: "[data-testid='email-field']"
    healed: "[data-testid='email-input']"  # сменили testid
    confidence: 0.92
    diff:
      - "data-testid: email-field → email-input"
      - "selector found by: signature match"
      
  - actionId: "action-4"
    original: "[data-testid='pay-btn']"
    healed: "[data-testid='checkout-pay']"
    confidence: 0.89
    diff:
      - "data-testid: pay-btn → checkout-pay"
      - "text: 'Pay Now' → 'Pay Now' (matched)"
```

### Step 5: Генерация Page Object из записи

```bash
# Генерируем TypeScript Page Object
frap generate \
  --scenario ./scenarios/payment-flow.yml \
  --lang typescript \
  --output ./pages/CheckoutPage.ts
```

Ожидаемый вывод:

```typescript
// Generated by frap from scenario: payment-flow
import { Page, Locator } from '@playwright/test';
import { withFrap } from '@frap/playwright';

export class CheckoutPage {
  constructor(private page: Page) {}
  
  // Element: email input
  private get emailInput() {
    return withFrap(this.page.locator('[data-testid="email-field"]'));
  }
  
  // Element: pay button  
  private get payButton() {
    return withFrap(this.page.locator('[data-testid="pay-btn"]'));
  }
  
  // Action: fill email
  async fillEmail(value: string) {
    await this.emailInput.fill(value);
  }
  
  // Action: click pay
  async clickPay() {
    await this.payButton.click();
  }
  
  // Composite: complete checkout
  async completeCheckout(email: string) {
    await this.fillEmail(email);
    await this.clickPay();
  }
}
```

### Step 6: Использование сгенерированного PO в тесте

```typescript
// test/checkout.spec.ts
import { test, expect } from '@playwright/test';
import { CheckoutPage } from '../pages/CheckoutPage';

test('payment flow', async ({ page }) => {
  const checkout = new CheckoutPage(page);
  
  await page.goto('http://localhost:3000/checkout');
  await checkout.completeCheckout('user@example.com');
  
  await expect(page.locator('[data-testid="success"]')).toBeVisible();
});
```

```bash
# Запуск теста
npx playwright test checkout.spec.ts

# Ожидаемый результат
✓ payment flow (4.2s)
```

### Step 7: Динамические элементы (React/Vue)

```bash
# Демо с динамически появляющимися элементами
docker-compose up -d checkout-dynamic

# Запись — элементы появляются по ходу
frap record \
  --url http://localhost:3000/checkout-dynamic \
  --output ./scenarios/dynamic-flow.yml

# Воспроизведение — MutationObserver отслеживает появление
frap replay \
  --scenario ./scenarios/dynamic-flow.yml \
  --wait-for-dynamic true
```

## Success Criteria

| Критерий | Ожидаемый результат |
|----------|-------------------|
| Запись | Сценарий сохраняется в YAML с полной структурой |
| Воспроизведение stable | healed=false, все action проходят |
| Healing | При смене testid находит элемент с confidence >= 0.85 |
| Генерация PO | TypeScript класс компилируется и работает |
| Динамические элементы | MutationObserver ловит появление элементов |

## Demo Script

```bash
#!/bin/bash
set -e

echo "=== C009: CDP Recording & Playback ==="

# Setup
cd demo-app
docker-compose up -d checkout-app

# Step 1: Record
echo "Recording scenario..."
frap record \
  --url http://localhost:3000/checkout \
  --output ./scenarios/payment-flow.yml \
  --name "payment-flow"

# Step 2: Replay stable
echo "Replaying on stable version..."
frap replay \
  --scenario ./scenarios/payment-flow.yml \
  --url http://localhost:3000/checkout

# Step 3: Replay with healing
echo "Switching to refactored version..."
docker-compose up -d checkout-v2

echo "Replaying with healing..."
frap replay \
  --scenario ./scenarios/payment-flow.yml \
  --url http://localhost:3000/checkout

# Step 4: Generate PO
echo "Generating Page Object..."
frap generate \
  --scenario ./scenarios/payment-flow.yml \
  --lang typescript \
  --output ./pages/

# Step 5: Run Playwright test
echo "Running generated test..."
cd pages && npx playwright test

echo "=== All steps completed ==="
```
