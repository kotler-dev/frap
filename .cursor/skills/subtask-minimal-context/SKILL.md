# Subtask with Minimal Context

## Purpose

Разбить большую задачу на подзадачи так, чтобы у каждой был небольшой набор файлов. Это экономит токены и улучшает точность работы агента.

## When to Use

- Пользователь просит разбить задачу на подзадачи
- Пользователь говорит «уменьши контекст» или «только эти файлы"
- Пользователь явно указывает «только @file1, @file2 в этой итерации"
- Задача касается большого числа файлов (>10)

## Workflow

1. **Анализ задачи**
   - Понять, какие части репо затронуты
   - Найти зависимости между файлами
   - Определить порядок реализации

2. **Разбиение на подзадачи**
   - Каждая подзадача — не более 5-7 файлов
   - Логическое завершение: компилируется, тестируется
   - Явный список файлов для каждой подзадачи

3. **Формат ответа**
   ```markdown
   ## Подзадача 1: [название]
   - **Цель**: что делаем
   - **Файлы**: @file1, @file2, @folder1
   - **Критерий готовности**: что проверяем

   ## Подзадача 2: [название]
   ...
   ```

4. **Предложение пользователю**
   - Показать список подзадач
   - Предложить выполнять по одной за раз
   - Дать команду для запуска первой подзадачи

## Rules

- **Не добавлять весь репо** для одного изменения
- **Не добавлять @project** без необходимости
- Если пользователь указал файл — не расширять контекст без явного разрешения
- Использовать `CONTEXT.md` и `project/` для архитектуры, не перечитывать десятки файлов
- Подзадачи должны быть **атомарными** (компилируются/тестируются отдельно)

## Example

**Задача**: «Реализовать F001 Self-Healing Selectors»

**Подзадачи**:

```markdown
## Подзадача 1: Signature extraction
- **Цель**: Реализовать извлечение сигнатуры DOM-элемента
- **Файлы**: 
  - @crates/signature/src/lib.rs (новый)
  - @crates/signature/Cargo.toml (новый)
- **Критерий готовности**: тесты на извлечение сигнатуры проходят

## Подзадача 2: Signature comparison
- **Цель**: Реализовать сравнение сигнатур и расчёт confidence
- **Файлы**:
  - @crates/signature/src/compare.rs
  - @crates/signature/src/lib.rs (обновить)
- **Критерий готовности**: юнит-тесты на сравнение проходят

## Подзадача 3: Healing orchestration
- **Цель**: Связать primary/fallback chain
- **Файлы**:
  - @crates/healing/src/lib.rs
  - @crates/frap-core/src/lib.rs (связка)
- **Критерий готовности**: CP001–CP003 проходят
```

## Output Format

Всегда возвращать:
1. Краткий анализ задачи (1-2 предложения)
2. Список подзадач с файлами
3. Предложение начать с конкретной подзадачи
4. Команду для пользователя (например, «Давай начнём с подзадачи 1»)
