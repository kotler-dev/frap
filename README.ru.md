# Frap

[![License](https://img.shields.io/badge/License-Apache%202.0-blue.svg)](LICENSE)

> Детерминированная библиотека для работы с локаторами в автоматизации тестирования.
> *Стабильные локаторы, когда интерфейс меняется.*

Frap помогает тестам переживать правки UI: находит элементы на странице, запоминает их структурный отпечаток и при необходимости подбирает новый селектор — с отчётом и **степенью уверенности** (confidence, шкала 0.0–1.0: насколько Frap уверен, что найденный элемент — тот же самый).

**Полный путь локатора:** карта страницы → отпечаток элемента → восстановление селектора → анализ причины падения → отслеживание изменений интерфейса между прогонами *(v1.2.0 — feedback loop, F009; v2.0+ — structural drift и CI gate, F017)*.

---

## Что такое Frap

**Что делает Frap:**

- **Находит** — за один проход строит карту интерактивных элементов с рекомендованными селекторами, без ручного обхода DOM
- **Удерживает** — при изменении разметки восстанавливает связь «селектор → элемент» по структуре, а не падает на первом же diff
- **Объясняет** — показывает степень уверенности, альтернативных кандидатов и причину падения, а не «магически починил»

Frap парсит деревья элементов (**DOM** — сейчас; **accessibility tree** — v1.4+ через CDP, v2.0+ мультиплатформа, F006), группирует похожие блоки детерминированными алгоритмами и генерирует стабильные локаторы для тестов.

---

## Проблема

Интерфейс меняется быстрее, чем обновляются тесты.

**Болевые точки:**

- Часы ручного разбора DOM на новых страницах
- Рефакторинг фронта вызывает массовые падения селекторов в CI
- Непонятно, почему автовосстановление выбрало новый элемент
- Непрозрачные облачные инструменты часто не проходят требования безопасности enterprise

**Чего ждут автоматизаторы:**

- Снять снимок страницы и получить все доступные локаторы
- Группировать элементы по структуре и бизнес-зонам
- Быстро стартовать PageObject без ручной рутины
- Снижать нестабильные падения через детерминированную логику

---

## Позиционирование

Frap — слой структуры и стабильных идентификаторов, а не замена ваших инструментов тестирования.

**Принцип: интеграция, а не замена.**

```
Playwright (сейчас) / Selenium·WebDriver (v1.4+) / JUnit
        ↓
TypeScript SDK / Java SDK / JSON-RPC
        ↓
Rust Core: отпечатки · кластеризация · восстановление · контекст
```

Frap оборачивает ваши существующие локаторы и добавляет структурную устойчивость.

---

## Жизненный цикл локатора

Один чёткий цикл для каждого локатора:

1. **Карта страницы** — собираем список элементов: теги, атрибуты, путь, роль, текст
2. **Отпечаток элемента** — фиксируем устойчивые признаки и структурный префикс (сигнатуру)
3. **Восстановление селектора** — если элемент не найден, ищем лучшего кандидата по степени уверенности и правилам выбора
4. **Отчёт** — что изменилось, какой селектор подобран, степень уверенности, топ кандидатов

## Основные возможности

### Карта страницы (Element Map)

За один проход Frap строит **карту страницы** — плоский каталог элементов с рекомендованным селектором и степенью уверенности. Это не дерево DOM и не `querySelector('*')`. Каталог — основа для PageObjects и baseline.

В API: `Frap.discover`, `buildElementMap`.

### Кластеризация

Элементы группируются по структурному шаблону:

- **SINGLE** — уникальный элемент (кнопка, поле)
- **LIST** — два и более похожих элемента (повторяющиеся карточки, строки)

Пять одинаковых карточек → один метод `items(index)`, а не пять копий. Кластеризация убирает шум и сужает поиск при восстановлении селектора.

### Восстановление локатора

Когда селектор перестал находить элемент (изменился `data-testid`, класс, обёртка), Frap **восстанавливает локатор** — подбирает новый селектор для того же элемента:

1. Сравнивает сохранённый отпечаток с текущей страницей
2. Ранжирует кандидатов по степени уверенности
3. Принимает решение: восстановить, если уверенность ≥ порога, или безопасно упасть с отчётом (например, два похожих кандидата — отказ, чтобы не кликнуть не туда)

Поведение детерминировано: одинаковый вход → одинаковый результат.

В API: `withFrap`, `heal`, `HealResult`.

### Сводный контекст и анализ причины падения

**Сводный контекст прогона** — единая временная шкала из трёх источников: действия в UI (клики, ошибки локаторов), сеть (HTTP, таймауты), логи (console, ошибки страницы).

**Анализ причины падения** — после падения теста Frap смотрит на этот контекст и отвечает: проблема в UI, в сети, в тайминге или неясна. Это **не** подбор нового селектора. Пример: API timeout → не «чинить» селектор, а указать сетевую причину.

В API: `captureAll`, `ContextTimeline`; `analyzeRca` → возвращает `RcaReport`.

### Генерация Page Object

Рабочий процесс от нуля до первых тестов:

- Карта страницы → фильтрация интерактивных элементов → группировка по кластерам → чтение рекомендованных селекторов → генерация скелета Page Object → развитие с baseline-отпечатками; отслеживание изменений UI между прогонами — **v1.2.0** (F009 feedback), **v2.0+** (F017 drift-report)

## Принципы

- **Объяснимость** — каждое решение показывает, как оно принято
- **Без ML по умолчанию** — детерминированные алгоритмы, одинаковый вход → одинаковый результат
- **Без облака** — работает без облачных API, on-prem по умолчанию
- **Интеграция, а не замена** — дополняет Playwright, Selenium и JUnit

---

## Архитектура

**Единое Rust-ядро**

- Единый источник истины для отпечатков, кластеризации и восстановления локаторов
- Одинаковое поведение в Java, TypeScript и будущих SDK
- Производительность и детерминизм без расхождений

**Тонкие SDK-адаптеры**

- JSON-RPC контракт и DTO-схема для всех транспортов
- Контрактные тесты для каждого SDK
- API-удобство без дублирования логики

Вы владеете своими отпечатками и можете воспроизвести поведение в любом стеке.

## Сводка API

Имена в таблице — целевой контракт Core/RPC; в конкретном SDK могут отличаться (например, `HealingEngine.heal()` в TypeScript). Матрица по поверхностям: [project/release/README.md](project/release/README.md).

| Действие | В API | Назначение |
|----------|-------|------------|
| Карта страницы | `Frap.discover`, `buildElementMap` | Снимок → каталог элементов |
| Фильтрация | `filterElementMap`, `FilterSpec` | Только интерактивные, мин. размер кластера |
| Генерация PO | `generatePageObject`, `GenerateOptions` | Скелет Page Object |
| Восстановление локатора | `withFrap`, `heal` → `HealResult` | Подбор нового селектора |
| Сводный контекст | `captureAll`, `ContextTimeline` | UI + сеть + логи |
| Анализ причины падения | `analyzeRca` → `RcaReport` | Классификация: UI / сеть / тайминг |
